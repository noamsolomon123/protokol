# Knesset OSINT Platform

**Objective, source-linked OSINT on Israeli politicians.** Every fact this
platform stores is pulled from a hard public source, carries a deep link back to
that source, and keeps the verbatim upstream payload. The platform never asserts
that a politician lied or is corrupt — it surfaces the *primary data* and *flags*
candidate contradictions for a human to review.

**Pilot scope:** Benjamin Netanyahu (`KNS_Person.Id = 965`) and the Likud party
(הליכוד / Likud). The schema and ingestion pipeline are built to scale from this
one MK to all 120 Knesset members and beyond (see
[docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)).

## 🌐 Live website

A mobile-first, Hebrew/RTL front-end is published via **GitHub Pages**:

**https://noamsolomon123.github.io/knesset-osint/**

It's a *static* site (`/docs`) that reads a JSON snapshot generated by
[`scripts/export_site_data.py`](scripts/export_site_data.py). It shows the
politician's profile, the **Accountability Scorecard + Index**, and their bills
and votes — every figure links to its official source. Rebuild the data with:

```bash
# uses a throwaway SQLite DB; no Postgres needed
DATABASE_URL=sqlite:///./_site_build.db python scripts/export_site_data.py --person-id 965
```

### Accountability Scorecard (the honest "score")

The site never prints a "good/bad" verdict. It shows a **transparent scorecard**
(`src/knesset_osint/scoring/`): each dimension is a ratio/count of sourced facts,
and the composite **Accountability Index** only folds in dimensions that are both
*available* and *objectively scorable*, reporting its **coverage** (e.g. "1 of 5
— preliminary"). Weights are explicit and adjustable — it's a *lens*, not a
verdict. Dimensions awaiting Phase-2 sources (integrity, promise-keeping,
finance/conflicts) are shown as "awaiting source", never guessed.

## Why this exists (inspiration)

This project is inspired by **Bruno César's `br-acc` / Aceleracionismo
Brasileiro** work — civic-tech that holds politicians accountable by putting
*structured, verifiable public data* in front of citizens instead of opinion.
We apply the same idea to the Knesset: pull the official record (votes, bills,
roles), link every datum to its source, and let the data speak.

---

## The objectivity mandate (non-negotiable)

These rules are the entire point of the platform. They are enforced in the data
model (see `db/mixins.py::ProvenanceMixin`) and in every mapper:

1. **Every record is source-linked.** Each stored row carries provenance:
   `source_type`, `source_name`, `source_url` (a deep link), `source_record_id`,
   the raw upstream `raw_payload`, and `fetched_at`. Evidence tables
   (`statements`, `actions`) enforce `source_url NOT NULL` at the DB level.
2. **Zero hallucination.** We never fabricate ids, dates, counts, or sources. If
   a value is missing upstream, we store `NULL`. Vote-table column names that are
   not pre-verified are probed live and read defensively.
3. **Contradictions are flagged, never asserted.** The verification layer creates
   `Contradiction` rows with status `needs_review` and **both** source links (the
   statement URL and the contradicting-evidence URL). A human reviewer sets
   `confirmed` / `dismissed`. The system never auto-labels anyone a liar.
4. **No secrets in code.** All config comes from environment variables via
   `core/config.py::settings`. Never hardcode passwords or tokens.

## Data sources

| Source | Role | Base URL (config field) | Notes |
|---|---|---|---|
| **ParliamentInfo** (OData **V4**) | source of truth | `settings.knesset_odata_v4_base` = `https://knesset.gov.il/OdataV4/ParliamentInfo` | Persons, bills, bill-initiators, positions. |
| **Votes.svc** (OData **V3**) | source of truth | `settings.knesset_votes_svc_base` = `https://knesset.gov.il/Odata/Votes.svc` | Vote headers + per-MK results. |
| **Open Knesset** (Hasadna) | enrichment only | `settings.open_knesset_pipelines_base`, `settings.knesset_data_gcs_base` | Best-effort, **never** source-of-truth. |
| Budget Key / Comptroller / Courts / Corporations Authority | future | — | Reserved `SourceType` enum members; see ARCHITECTURE.md. |

### ⚠️ Votes V4 is bot-protected — we use V3 `.svc`

The **OData V4 Votes** endpoint sits behind an **Imperva bot challenge** (it
returns `200` + an HTML challenge page instead of JSON). We do **not** add any
code to defeat that challenge. Instead we use the clean, open official **V3
`Votes.svc`** endpoint. The shared `ODataClient._get` is defensive: if a
response is HTML instead of JSON it raises a retryable error rather than crashing
the parser.

**Column-name caveat:** the exact columns of the V3 vote tables
(`View_vote_rslts_hdr_Approved`, `vote_rslts_kmmbr_shadow`,
`View_Vote_MK_Individual`, `vote_result_type`) are **not** pre-verified. Probe
them live before relying on them, e.g.:

```bash
curl -s "https://knesset.gov.il/Odata/Votes.svc/View_Vote_MK_Individual?\$top=1&\$format=json"
```

Mappers read these rows defensively with `row.get()` over multiple candidate keys.

## Quickstart (Docker — recommended)

Prerequisites: Docker + Docker Compose.

```bash
# 1. Configure. Copy the template and edit secrets (Neo4j password, etc.).
cp .env.example .env
#    -> set NEO4J_PASSWORD; leave the verified source URLs as-is.

# 2. Start PostgreSQL + Neo4j (and the API, if the compose file ships one).
docker compose up -d

# 3. Apply the database schema.
docker compose run --rm api alembic upgrade head
#    (or run alembic locally against the compose Postgres — see local path below)

# 4. Ingest the pilot politician (Netanyahu, KNS_Person.Id = 965).
knesset-osint ingest --person-id 965

# 5. Open the interactive API docs.
#    http://localhost:8000/docs
```

The `knesset-osint` console command is the project's CLI (declared in
`pyproject.toml` as `knesset-osint = "knesset_osint.cli:app"`). Run
`knesset-osint --help` to list subcommands.

## Verifying it worked

After `ingest --person-id 965` you should have one `politicians` row
(`knesset_person_id = 965`, `last_name = נתניהו`), Netanyahu's roles from
`KNS_PersonToPosition`, his ~31 bill-sponsorship rows from `KNS_BillInitiator`,
and his per-MK votes reconciled from `View_Vote_MK_Individual`. Quick check:

```bash
knesset-osint show --person-id 965        # summary counts per source
# or hit the API: GET http://localhost:8000/politicians/965
```

If counts are zero, confirm the live endpoints are reachable (the `curl` probe
above) and that `.env` was loaded.

```
  db/            Base, engine/session, Timestamp + Provenance mixins

alembic/         migrations (env + versions/)
docs/            ARCHITECTURE, ADDING_A_POLITICIAN, ADDING_A_SOURCE
tests/           pytest suite (respx for HTTP mocking)
```
